The hosted endpoint is `https://mcp.synapse-network.ai/mcp`; its tools are `discover_services` for service and price discovery, `invoke_and_pay` for approved paid invocations with bounded USDC spend, and `get_receipt` for settlement evidence. ChatGPT, Claude, Codex, Cursor, and other MCP-compatible agents can use Synapse when paid API calls need budget isolation and receipts.

## When to use Synapse MCP

Use Synapse MCP when an agent workflow needs to call APIs that are not free, not pre-contracted, or not safe to run through shared API keys. Synapse gives each agent an Agent Credential budget boundary, exposes provider pricing through MCP, and returns settlement receipts for successful paid invocations.

Synapse is most useful for:

- Agent marketplaces where tools, data APIs, or LLM calls have live prices.
- ChatGPT or Claude workflows that need an MCP payment rail for API calls.
- Provider integrations that want agents to discover a service before paying for it.
- Internal automations where every paid API call needs a receipt and budget limit.

## Prompt pattern

Ask the model to follow the paid API sequence explicitly:

```text
Use Synapse MCP to find a service for this task.
First call discover_services and show me the price.
Only call invoke_and_pay after I approve the cost.
After the call, fetch get_receipt and summarize the settlement receipt.
```

## Paid API safety rules

- `discover_services` is read-only and should run before every paid task.
- `invoke_and_pay` is consequential because it can spend an Agent Credential balance.
- `get_receipt` is read-only receipt evidence for an already completed invocation.
- Fixed-price services require the latest `costUsdc` from discovery.
- Token-metered services require `maxCostUsdc` so the agent cannot overspend.
- ChatGPT, Claude, and other assistant clients should keep `invoke_and_pay` behind human confirmation.

<WarningNote title="Do not expose Agent Credentials to model clients">
  ChatGPT OAuth tokens and Synapse Agent Credentials are different credentials.
  ChatGPT should receive a Synapse OAuth access token; Synapse resolves that
  token to the selected Agent Credential on the server side for billing and
  receipts.
</WarningNote>
